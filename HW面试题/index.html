# Windows 入侵排查思路？

  2. 分析进程：检查所有正在运行的进程，查看是否有可疑的进程或进程行为，比如 CPU 占用率高、网络流量异常等。

# Linux 入侵排查的常见思路：

  1. 收集信息：收集有关系统和网络的信息，包括系统日志、安全日志、进程列表、网络连接列表、服务列表等。
  2. 分析进程：检查所有正在运行的进程，查看是否有可疑的进程或进程行为，比如 CPU 占用率高、网络流量异常等。
  3. 检查文件：检查系统文件是否被修改或替换，比如检查 /etc/passwd、/etc/shadow 等文件是否被篡改。
  4. 检查网络连接：查看是否有未知的网络连接或远程连接，比如远程 SSH 连接、FTP 连接等。
  5. 检查系统服务：检查系统服务是否被修改或替换，比如检查是否有未知的系统服务或后台进程。
  6. 分析日志：分析系统日志和安全日志，查找异常的登录尝试、文件访问等行为。
  7. 检查用户账户：检查系统中的用户账户，查看是否有未知的用户或用户行为。
  8. 更新系统和软件：及时更新系统和软件，修补已知的漏洞，提高系统的安全性。

# webshell 绕过的方法

文件名混淆：将 webshell 文件名中的关键字进行混淆，如将 "shell.php" 改为 "shll.php" 等，使其不易被检测到。

文件内容加密：将 webshell 文件内容进行加密，使其不易被检测到。可以使用加密算法如 AES、DES 等进行加密。

隐藏文件属性：将 webshell 文件属性设置为隐藏，使其不易被发现。可以使用命令行或文件管理器进行设置。

伪造文件类型：将 webshell 文件的扩展名改为其他文件类型，如将 ".php" 改为 ".jpg" 等，使其不易被检测到。

利用文件包含漏洞：通过文件包含漏洞，将 webshell 代码插入到目标网站的正常代码中，使其不易被检测到。

利用上传漏洞：通过上传漏洞，将 webshell 文件上传到目标网站中，使其不易被检测到。

利用反弹 shell：通过反弹 shell，从目标服务器上执行命令，不需要在目标服务器上存放 webshell，从而避免被检测到。

利用加密传输：通过加密传输，将 webshell 传输到目标服务器，使其不易被检测到。可以使用加密协议如 SSL、TLS 等进行加密传输。

# 应急响应

应急响应是指在网络安全事故发生时，及时采取措施，快速响应并解决问题，以保护企业的资产和业务正常运转。以下是应急响应的几个步骤：

  1. 预案制定：在网络安全事故发生前，制定完善的应急响应预案，包括预警机制、应急响应流程、人员职责及联系方式等，以应对突发事件。
  2. 事故发现：及时发现网络安全事故，通过监测工具、日志分析等方式，对异常行为进行分析，判断是否为安全事件。
  3. 事件调查：对安全事件进行调查，收集证据，分析攻击手段和攻击路径，确定受影响的系统和数据，并进行风险评估。
  4. 事件处理：根据调查结果，采取相应的应急响应措施，包括隔离受影响的系统和数据、修复漏洞、清除恶意代码等。
  5. 恢复业务：在安全事件得到控制后，进行系统恢复和数据恢复，确保业务正常运转。
  6. 事件总结：对安全事件进行总结和分析，总结经验教训，完善应急
